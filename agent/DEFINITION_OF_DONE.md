# Definition of Done

A phase is done only when:

- the intended user flow works end-to-end at the scaffold level
- the code compiles or the gap is explicitly documented as environmental
- task board statuses are current
- work log is updated
- docs reflect the actual repo state
- there is no hidden architectural drift

## MVP done criteria

- user can open Prompt Studio in VS Code
- user can input a brief and brand profile
- user can compile multiple target outputs
- user can export a structured bundle into the workspace
- the same compiler path is callable via CLI
- API scaffold exposes compile and health routes
- example files are valid and useful
